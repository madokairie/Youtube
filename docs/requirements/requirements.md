# SEO Research & Article Production App - 統合要件定義書

**プロジェクト名**: SEO Article Factory
**バージョン**: 0.1.0
**作成日**: 2026-03-06
**対象ユーザー**: 円香さん（マーケブログ + 美容ブログ運営者）

---

## 1. プロジェクト概要

### 1.1 目的

SEOリサーチから記事生成・クロスプラットフォーム展開までをワンストップで実現するアプリケーション。
**最終ゴール**: SEO上位表示 → リスト獲得（LINE登録等） → 成約

### 1.2 対象ブログ

| ブログ | ジャンル | 主要ターゲット |
|--------|---------|---------------|
| マーケブログ | マーケティング・ビジネス | BtoB事業者・マーケター |
| 美容ブログ | 美容・スキンケア・コスメ | 20〜40代女性 |

### 1.3 コアフロー

```
自動分析（SC/GA）→ 競合分析 → ギャップ発見 → 記事提案
→ 円香さんが選択 → 記事構成生成 → 本文生成 → SEO最適化
→ クロスプラットフォーム展開 → 順位トラッキング → リライト提案
```

## 2. 機能要件

### FR-001: Google Search Console 自動連携

- **優先度**: Critical
- **説明**: Google OAuth認証でSearch Consoleに接続し、2ブログのデータを自動取得・分析
- **受入基準**:
  - [ ] Google OAuth 2.0 認証フローが動作する
  - [ ] 2つのプロパティ（マーケブログ・美容ブログ）を切り替えられる
  - [ ] 以下のデータを自動取得:
    - 検索クエリ別のクリック数・表示回数・CTR・掲載順位
    - ページ別パフォーマンス
    - デバイス別・国別データ
  - [ ] 過去90日分のデータをダッシュボードに表示
  - [ ] 週次・月次のトレンド分析を自動生成

### FR-002: Google Analytics 自動連携

- **優先度**: Critical
- **説明**: GA4 Data APIに接続し、2ブログのユーザー行動を自動分析
- **受入基準**:
  - [ ] GA4プロパティに接続できる
  - [ ] 以下のデータを自動取得:
    - PV数・UU数・セッション数
    - 流入元（オーガニック・SNS・ダイレクト）
    - ランディングページ別の直帰率・滞在時間
    - コンバージョン（CTA クリック率）
  - [ ] 記事別のパフォーマンスランキングを表示

### FR-003: 競合分析エンジン

- **優先度**: Critical
- **説明**: 指定キーワードの検索上位10記事をスクレイピングし、構成・共起語・文字数を分析
- **受入基準**:
  - [ ] SerpAPI経由で検索上位10件のURLを取得
  - [ ] 各記事から以下を抽出:
    - 見出し構成（H1/H2/H3階層）
    - 本文の文字数
    - 共起語・関連語（TF-IDF分析）
    - メタディスクリプション
    - 内部リンク数・外部リンク数
    - 構造化データの有無（FAQ等）
  - [ ] 上位記事の共通パターンをAIが要約
  - [ ] 「上位記事が網羅しているが自分の記事にないトピック」を検出

### FR-004: AIギャップ分析 & 記事提案

- **優先度**: Critical
- **説明**: SC/GAデータと競合分析の結果を突合し、円香さんが書くべき記事をリストアップ
- **受入基準**:
  - [ ] 以下の観点でギャップを検出:
    - 検索ボリュームがあるが記事がないキーワード
    - 順位が11〜30位で伸びしろがあるキーワード
    - 競合が書いているが自分にない記事
    - 季節性・トレンドキーワード
  - [ ] 提案リストに以下を表示:
    - 推定検索ボリューム
    - 競合の強さ（ドメインパワー目安）
    - 想定難易度（Easy/Medium/Hard）
    - リスト獲得ポテンシャル（High/Medium/Low）
  - [ ] 円香さんがリストから「書きたい記事」を選択するUI
  - [ ] 選択した記事はキューに追加される

### FR-005: 記事構成自動生成

- **優先度**: Critical
- **説明**: 選択したキーワードに対して、SEO最適化された記事構成を自動生成
- **受入基準**:
  - [ ] 以下の構成テンプレートで生成:
    ```
    H1: [メインキーワード含むタイトル]
      導入（読者の悩み共感 + この記事で分かること）
      H2: [サブトピック1]
        H3: [詳細1-1]
        H3: [詳細1-2]
      H2: [サブトピック2]
        H3: [詳細2-1]
        H3: [詳細2-2]
      H2: [サブトピック3]
        H3: [詳細3-1]
        H3: [詳細3-2]
      H2: まとめ
        要点整理 + CTA
    ```
  - [ ] 各H2/H3に「書くべき内容の要約」を付与
  - [ ] 競合上位記事の共起語を構成に反映
  - [ ] 構成は編集可能（H2/H3の追加・削除・並び替え）
  - [ ] 構成承認ボタンで本文生成に進む

### FR-006: 本文生成エンジン

- **優先度**: Critical
- **説明**: 承認された構成に基づき、3000〜5000文字のSEO最適化された本文を生成
- **受入基準**:
  - [ ] Claude APIを使用して本文を生成
  - [ ] 文字数: 3000〜5000文字
  - [ ] 以下のSEOルールを自動適用:
    - 冒頭300文字にメインキーワードを含める
    - 共起語を自然に散りばめる（過剰な詰め込みは避ける）
    - 各H2/H3の直下にキーワードを含む文を配置
    - 読みやすい文章構造（一文60文字以内推奨）
  - [ ] E-E-A-T強化テンプレートを適用（後述 FR-012）
  - [ ] 生成後にリッチエディタで編集可能
  - [ ] 「再生成」ボタンで部分的な再生成が可能（H2単位）

### FR-007: メタディスクリプション & タイトル生成

- **優先度**: High
- **説明**: SEO最適化されたメタディスクリプションとタイトル候補を生成
- **受入基準**:
  - [ ] メタディスクリプション:
    - 120〜160文字
    - メインキーワードを含む
    - CTRを高めるコピーライティング（数字・疑問形・ベネフィット）
    - 3候補から選択可能
  - [ ] タイトル候補:
    - 5候補を生成
    - パターン: 数字入り / 疑問形 / How to / 比較型 / 体験型
    - 各候補に想定CTRスコアを表示
    - 32文字以内（検索結果での省略防止）

### FR-008: CTA自動挿入

- **優先度**: High
- **説明**: 記事中にリスト獲得用CTAを自動挿入
- **受入基準**:
  - [ ] CTA挿入ポイント:
    - 記事中盤（2つ目のH2の後）
    - 記事末尾（まとめセクション内）
  - [ ] CTAタイプを設定可能:
    - LINE公式アカウント登録
    - メルマガ登録
    - 無料相談予約
    - 資料ダウンロード
  - [ ] ブログ別にCTA設定を保持
  - [ ] CTAのテキスト・リンク先を編集可能

### FR-009: 内部リンク提案

- **優先度**: High
- **説明**: 既存記事への内部リンクを自動提案し、サイト回遊率とSEO評価を向上
- **受入基準**:
  - [ ] 既存記事のタイトル・キーワード・URLをDBに保持
  - [ ] 新記事生成時に関連する既存記事を自動検出
  - [ ] 挿入推奨箇所とアンカーテキストを提案
  - [ ] 1記事あたり3〜5本の内部リンクを目安
  - [ ] 承認制（自動挿入ではなく提案→承認）

### FR-010: リード導線マッピング

- **優先度**: Medium
- **説明**: 「この記事を読む人 → 次に読むべき記事 → CTA」の導線を設計
- **受入基準**:
  - [ ] 記事間の関連性マップを可視化
  - [ ] 検索意図の段階（認知→興味→検討→行動）に沿った導線提案
  - [ ] 各記事の「次に読む記事」を自動設定
  - [ ] コンバージョンまでの記事ステップ数を表示

### FR-011: プラットフォーム横展開

- **優先度**: High
- **説明**: ブログ記事をNote・X（長文）・X（ポスト）に変換。カニバリ防止のため媒体特性に合わせてトーン・切り口・文字数を変換
- **受入基準**:
  - [ ] 以下の変換ルールで展開:

    | 元記事 | 展開先 | 変換ルール |
    |--------|--------|-----------|
    | ブログ(3000-5000文字) | Note | 体験談・ストーリー寄りにリライト、2000-3000文字、一人称視点 |
    | ブログ(3000-5000文字) | X長文 | 要点凝縮・導入フック強め、280-1000文字、箇条書き活用 |
    | ブログ(3000-5000文字) | Xポスト | 1記事から3-5投稿分のフック生成、各140文字以内 |

  - [ ] カニバリ防止ルール:
    - 各プラットフォームで検索意図・読者層・トーンを明確に変える
    - ブログ: 網羅的・専門的（検索流入狙い）
    - Note: 体験ベース・共感型（Note内検索+SNS拡散狙い）
    - X: フック・議論喚起型（エンゲージメント狙い）
    - 同一文章のコピペ禁止（類似度チェック機能）
  - [ ] 展開タイミングの提案（ブログ公開→3日後にNote→1週間後にX等）
  - [ ] 各プラットフォームのプレビュー表示

### FR-012: E-E-A-T強化テンプレート

- **優先度**: High
- **説明**: Google E-E-A-T（Experience, Expertise, Authoritativeness, Trustworthiness）を強化するテンプレートを本文に自動適用
- **受入基準**:
  - [ ] 以下のE-E-A-T要素を記事に組み込む:

    | 要素 | 実装方法 |
    |------|---------|
    | Experience（経験） | 「実際に〇〇を試した結果」「私のクライアントの事例では」等の体験ベース文言 |
    | Expertise（専門性） | 専門用語の適切な使用と解説、データ・統計の引用 |
    | Authoritativeness（権威性） | 著者プロフィール、実績数値、メディア掲載歴 |
    | Trustworthiness（信頼性） | 情報ソースの明記、最終更新日、免責事項 |

  - [ ] ブログ別のE-E-A-Tプロフィール設定:
    - マーケブログ: マーケティング専門家としてのプロフィール
    - 美容ブログ: 美容の実体験者としてのプロフィール

### FR-013: FAQセクション（構造化データ対応）

- **優先度**: High
- **説明**: 記事末尾にFAQセクションを生成し、JSON-LD構造化データを出力
- **受入基準**:
  - [ ] キーワードに関連するFAQを3〜5件自動生成
  - [ ] FAQ Schema（JSON-LD）を自動出力
  - [ ] Googleリッチリザルトでの表示を想定した質問・回答形式
  - [ ] FAQは編集・追加・削除可能

### FR-014: 順位トラッキング & リライト提案

- **優先度**: Medium
- **説明**: 公開記事の検索順位を自動追跡し、順位改善のためのリライト提案を生成
- **受入基準**:
  - [ ] Search Console APIから順位データを日次取得
  - [ ] 以下の条件でリライト提案をトリガー:
    - 公開後30日経過で順位30位以下
    - 順位が2週間連続で下降
    - CTRが同順位帯の平均を下回る
  - [ ] リライト提案の内容:
    - 追加すべきキーワード・共起語
    - 構成の改善案
    - 競合との差分ポイント
  - [ ] リライト優先度でソート（インパクト順）

### FR-015: カニバリ検出

- **優先度**: High
- **説明**: ブログ内およびプラットフォーム間でのキーワードカニバリゼーションを検出
- **受入基準**:
  - [ ] ブログ内カニバリ:
    - 同じキーワードで複数記事が競合している場合を検出
    - 統合・リダイレクト・差別化の提案
  - [ ] プラットフォーム間カニバリ:
    - ブログとNoteで同じ検索クエリに表示されている場合を検出
    - 各プラットフォームの役割分担を提案
  - [ ] カニバリスコア表示（重複度 0〜100%）
  - [ ] 新記事作成時に既存記事とのカニバリ事前チェック

### FR-016: ダッシュボード

- **優先度**: High
- **説明**: 2ブログの概要・アクション推奨を一覧表示するメイン画面
- **受入基準**:
  - [ ] ブログ切り替えUI（マーケブログ / 美容ブログ）
  - [ ] 表示項目:
    - 今月のPV・UU・オーガニック流入数
    - 順位変動サマリー（上昇/下降/新規ランクイン）
    - 未対応アクション数（リライト提案・新記事提案）
    - 記事生成キュー（選択済み未着手記事）
  - [ ] ワンクリックでアクション画面へ遷移

## 3. ナレッジテンプレート（プロンプト組み込み型）

アプリ内DBではなく、AI生成時のプロンプトに組み込む形で円香さんの専門知識を活用。

### 3.1 マーケブログ用ナレッジ

```
## 著者コンテキスト（マーケブログ）
- マーケティング専門家として執筆
- BtoB・BtoCの両方のマーケティング経験あり
- SNSマーケティング、コンテンツマーケティング、SEOに精通
- 実務経験に基づく具体的なアドバイスを得意とする
- データドリブンなアプローチを重視

## トーン・スタイル
- 専門的だが分かりやすい
- 「〜すべき」ではなく「〜がおすすめ」の柔らかい語調
- 具体的な数字・事例を多用
- 読者に寄り添う姿勢

## よく使う専門知識
- CVR改善、LTV最大化、ファネル設計
- SNS運用（Instagram、X、LINE）
- コンテンツマーケティング戦略
- Google Analytics / Search Console の活用法
```

### 3.2 美容ブログ用ナレッジ

```
## 著者コンテキスト（美容ブログ）
- 美容・スキンケアの実体験者として執筆
- 実際に試した商品のリアルなレビューを重視
- 成分知識に基づく科学的な解説も取り入れる
- 読者と同じ目線で悩みを共有

## トーン・スタイル
- 親しみやすいが信頼できる
- 一人称「私」を使用
- ビフォーアフターや使用感を詳細に記述
- 「〜してみてね」「〜だよ」の親近感ある語調

## よく使う専門知識
- スキンケア成分（セラミド、レチノール、ナイアシンアミド等）
- 肌タイプ別のアドバイス
- 季節別のケア方法
- プチプラ〜デパコスの幅広い製品知識
```

### 3.3 プラットフォーム別トーン変換ルール

```
## ブログ → Note 変換テンプレート
- 視点: 網羅的解説 → 個人の体験ストーリー
- トーン: 専門的 → 共感・エモーショナル
- 構成: H1/H2/H3体系 → エッセイ・ストーリー形式
- 導入: 「〇〇について解説します」→「〇〇で悩んでいた私が見つけた答え」
- CTA: 直接的 → さりげない誘導

## ブログ → X長文 変換テンプレート
- 視点: 詳細解説 → 要点のみ凝縮
- トーン: 丁寧 → 端的・インパクト重視
- 構成: 段落形式 → 箇条書き・改行多用
- 導入: フックから始める（意外性・数字・問いかけ）
- CTA: 「詳しくはブログで→[URL]」

## ブログ → Xポスト 変換テンプレート
- 1記事 → 3〜5投稿に分解
- パターン: 問題提起 / 驚きの事実 / Tips / 実体験 / まとめ
- 各140文字以内
- ハッシュタグ2〜3個付与
- スレッド形式も選択可能
```

## 4. 非機能要件

### NFR-001: パフォーマンス

| 項目 | 目標値 |
|------|--------|
| ダッシュボード表示 | 2秒以内 |
| 競合分析（10記事） | 30秒以内 |
| 記事構成生成 | 10秒以内 |
| 本文生成（3000-5000文字） | 60秒以内 |
| プラットフォーム変換（3媒体） | 30秒以内 |

### NFR-002: セキュリティ

- Google OAuth トークンの安全な管理（Supabase Auth）
- API キーの環境変数管理
- HTTPS必須
- CSRFトークン
- Rate Limiting（API呼び出し制限）

### NFR-003: スケーラビリティ

- 記事数: 1ブログあたり1,000記事まで対応
- 同時分析: 5キーワードまで並列処理
- データ保持: 順位データ365日分

### NFR-004: ユーザビリティ

- レスポンシブ対応（デスクトップ優先、タブレット対応）
- ダークモード対応
- 日本語UI
- ローディング状態の明示（プログレスバー）

## 6. ユーザーストーリー

### US-001: ブログ現状分析
**As** 円香さん
**I want** 2つのブログのSCとGAデータを自動分析してほしい
**So that** 現状のパフォーマンスを一目で把握し、改善ポイントが分かる

### US-002: 書くべき記事の発見
**As** 円香さん
**I want** AIが競合分析とギャップ分析をして「書くべき記事」をリストアップしてほしい
**So that** キーワード調査の手間を省き、効果の高い記事に集中できる

### US-003: 記事選択と構成生成
**As** 円香さん
**I want** 提案リストから書きたい記事を選んで、SEO最適化された構成を自動生成してほしい
**So that** 記事の骨組みを考える時間を節約できる

### US-004: 本文生成
**As** 円香さん
**I want** 構成に基づいて3000-5000文字の本文を自動生成してほしい
**So that** 記事作成の時間を大幅に短縮し、量産できる

### US-005: クロスプラットフォーム展開
**As** 円香さん
**I want** ブログ記事をNote・X向けにカニバリなく変換してほしい
**So that** 1つの記事から複数プラットフォームでリーチを最大化できる
